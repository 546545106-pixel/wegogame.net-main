# 快速部署游戏指南

## 🚀 最简单的方法

### 步骤1：双击运行批处理文件

直接双击项目根目录下的 **`部署游戏.bat`** 文件

脚本会自动：
- ✅ 解压zip文件
- ✅ 查找所有游戏
- ✅ 复制到 `public/games/` 目录
- ✅ 自动添加到游戏列表

### 步骤2：启动开发服务器

```bash
npm run dev
```

### 步骤3：访问网站查看新游戏

打开浏览器访问 `http://localhost:5173`，新游戏会自动显示在游戏列表中。

---

## 📋 如果批处理文件无法运行

### 方法A：手动运行Python脚本

1. 打开命令提示符（CMD）或PowerShell
2. 进入项目目录：
   ```bash
   cd "d:\游戏网站1\wegogame.net-main"
   ```
3. 运行脚本：
   ```bash
   python 部署游戏.py
   ```

### 方法B：手动部署（如果Python不可用）

请参考 **`游戏部署指南.md`** 文件中的"方法二：手动部署"部分。

---

## ⚠️ 注意事项

1. **游戏URL配置**
   - 脚本会自动使用相对路径 `/games/游戏名/index.html`
   - 这样在开发和生产环境都能正常工作

2. **游戏缩略图**
   - 新游戏会使用默认缩略图
   - 您可以后续手动编辑 `src/data/games.js` 更新缩略图URL

3. **游戏信息**
   - 脚本会自动生成游戏标题、描述等信息
   - 您可以后续手动编辑 `src/data/games.js` 优化游戏信息

4. **备份文件**
   - 原 `games.js` 文件会自动备份为 `games.js.backup`
   - 如果需要恢复，可以重命名备份文件

---

## 🔍 验证部署

部署完成后，检查以下内容：

1. ✅ `public/games/` 目录下是否有游戏文件夹
2. ✅ `src/data/games.js` 文件是否已更新
3. ✅ 运行 `npm run dev` 后游戏是否显示
4. ✅ 点击游戏后是否能正常加载和运行

---

## 📝 后续优化建议

1. **添加游戏缩略图**
   - 为每个游戏准备512x512的缩略图
   - 更新 `games.js` 中的 `thumb` 字段

2. **优化游戏信息**
   - 编辑 `games.js` 更新游戏标题、描述、分类等
   - 添加合适的标签（tags）

3. **分类管理**
   - 根据游戏类型设置合适的 `category`
   - 常见分类：Arcade, Puzzle, Action, Adventure, Racing, Sports, Strategy, Casual

4. **星级评分**
   - 根据游戏质量调整 `star` 值（1-5）

## ❓ 遇到问题？

1. **游戏无法加载**
   - 检查 `public/games/` 目录下是否有游戏文件
   - 检查浏览器控制台是否有错误
   - 确认游戏URL路径正确

2. **脚本运行失败**
   - 确认Python已安装（运行 `python --version` 检查）
   - 确认zip文件路径正确
   - 查看错误信息并参考 `游戏部署指南.md`

3. **游戏显示但无法运行**
   - 检查游戏HTML文件是否存在
   - 检查游戏资源文件是否完整
   - 查看浏览器控制台错误信息

---
